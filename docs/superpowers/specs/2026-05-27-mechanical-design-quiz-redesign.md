# 机械设计基础刷题器美化与功能增强设计

## 背景

当前项目只有一个 Python 文件 `r3pxcolg.py`，其中同时包含题库数据和 Tkinter 刷题界面。题库约 512 道题，现有程序支持随机出题、答错加入错题队列、鼠标选择和键盘 `A/B/C/D` 答题、`Enter` 下一题。实施时将把这个主文件直接重命名为可读的 `mechanical_design_quiz.py`。

本轮目标是在保持桌面程序形态的前提下，美化界面并增强刷题体验。改动必须外科式，避免修改与本轮需求无关的题库内容。

## 目标

- 保留 Tkinter 桌面程序，不迁移为 Web 应用。
- 将题库数据和界面逻辑轻量拆分，降低后续维护难度。
- 参考用户提供的网页版刷题器布局：顶部标题、左侧题号导航、中间答题区、右上设置入口、可切换设置面板。
- 增加顺序/随机出题模式。
- 增加左侧题号跳转。
- 增加自动保存、手动保存、加载进度和重置进度。
- 增加答错自动错题标记，并支持手动标记/取消标记。
- 保留并扩展键盘操作。

## 非目标

- 不重写为浏览器 Web 应用。
- 不整理章节、不给题库手工补章节名称。
- 不人工修改题目、选项、答案、解析文本。
- 不引入大型框架或复杂依赖。
- 不实现账号、多用户、云同步。

## 文件结构

采用轻拆分方案：

- `mechanical_design_quiz.py`：由原 `r3pxcolg.py` 直接重命名而来，作为新的主程序入口，包含 Tkinter UI、答题流程、键盘事件、设置面板逻辑。
- `questions_data.py`：只保存 `questions = [...]` 题库数据。题目内容从原文件迁移，保持原样。
- `quiz_progress.json`：运行时自动生成的进度文件，保存刷题状态。
- `docs/superpowers/specs/2026-05-27-mechanical-design-quiz-redesign.md`：本设计文档。

## UI 布局

主窗口默认显示两栏：

- 顶部栏：应用标题“机械设计基础刷题器”、提示文本“仅供学习 请勿商用”、保存按钮、设置按钮。
- 左侧栏：单一题库题号网格，显示 `1-512`。不做章节分组。
- 中间答题区：显示当前模式、题号进度、保存状态、题干、选项、操作按钮、解析区。
- 设置面板：默认隐藏。点击右上角“设置”按钮后打开，点击“确定/关闭”后收起。

题号状态建议用颜色区分：

- 未答：默认浅色。
- 当前题：蓝色高亮。
- 已答正确：绿色。
- 已答错误或自动错题：红色。
- 手动标记：额外标记色，例如橙色下划线或角标。

## 答题交互

选择答案和判题分离：

- 点击选项或按 `A/B/C/D` 只选择答案，不立即判题。
- 点击“检查”或按 `Enter` 执行判题。
- 判题后显示正确答案和解析。
- 判题后再次按 `Enter` 进入下一题。
- `←` 跳到上一题。
- `→` 跳到下一题。

按钮行为：

- “检查”：未检查时可用；检查后可保持可见但不重复改变结果。
- “下一题”：进入当前模式下的下一题。
- “标记错题/取消标记”：切换手动标记状态。

## 出题模式

支持两种模式：

- 顺序模式：下一题按原始题号递增。
- 随机模式：生成并保存一套随机顺序，下一题按随机队列前进。

左侧题号始终代表原始题号。随机模式下点击左侧题号 `13`，仍跳到原始第 13 题，而不是随机队列中的第 13 个位置。

切换模式时不清空答题状态。随机顺序应保存到进度文件中，避免关闭重开后随机队列变化导致学习进度混乱。

## 错题与标记

错题状态包含两个来源：

- 自动错题：答错后自动加入。
- 手动标记：用户主动标记或取消。

界面上可以统一显示为错题/标记状态，但内部应区分来源，避免取消手动标记时误清除答错记录。

建议保存字段：

- `auto_wrong`: 是否曾经答错。
- `manual_marked`: 是否手动标记。
- `is_correct`: 最近一次检查是否正确。
- `checked`: 是否已检查。
- `selected`: 用户选择的答案。

## 进度保存

进度文件为 `quiz_progress.json`，默认自动保存开启。保存内容包括：

- 当前题原始索引。
- 当前模式：`sequential` 或 `random`。
- 随机顺序列表。
- 自动保存开关。
- 每道题的选择答案。
- 每道题是否已检查。
- 每道题是否答对。
- 每道题是否自动错题。
- 每道题是否手动标记。

自动保存触发时机：

- 选择答案后。
- 检查答案后。
- 手动标记或取消标记后。
- 切换题目后。
- 切换顺序/随机模式后。
- 点击“保存进度”后。

手动操作：

- “保存当前进度”：立即写入进度文件。
- “加载之前的进度”：读取 `quiz_progress.json` 并刷新界面。
- “重置进度”：二次确认后清空答题状态、错题标记和当前题位置，并重新生成随机顺序。

## 异常处理

- 题库导入失败：弹窗提示题库不可用，程序不继续进入答题。
- 进度文件不存在：创建默认进度，从第一题开始。
- 进度文件损坏或格式不兼容：提示“进度不可用，已从头开始”，并使用默认进度。
- 重置进度：必须二次确认。
- 保存失败：提示保存失败原因，答题界面保持可用。

## 实现约束

- 题库文本迁移时保持原样，不做人工纠错。
- 避免引入额外第三方依赖。
- 不保留旧文件名兼容入口；旧主文件直接改名为 `mechanical_design_quiz.py`。
- UI 改动集中在新的主程序文件中。
- 进度文件作为运行时产物，不参与题库源数据。
